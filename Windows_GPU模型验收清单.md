# Windows GPU 模型验收清单

这份清单用于在 Windows + GPU 机器上判断一版模型是否达到可交付水平。目标不是“能出声”，而是确认是否已经接近最佳唱歌效果。

## 1. 验收前准备

确认以下文件已就绪：

1. 主模型：`logs/44k/G_*.pth`
2. 主配置：`logs/44k/config.json`
3. 音质增强模型：`logs/44k/diffusion/model_0.pt`
4. 音质增强配置：`logs/44k/diffusion/config.yaml` 或对应配置路径
5. 音色增强文件：`logs/44k/feature_and_index.pkl`

确认以下预设一致：

1. 使用 [quality_presets.py](/Volumes/D/MyGitHub/so-vits-svc-icee/quality_presets.py) 的默认值
2. WebUI 质量模式选 `极致质量`
3. 如无特殊需要，只调整 `变调`

## 2. 测试素材要求

建议准备 3 组素材，每组 15 到 30 秒：

1. 中音区稳定旋律
2. 高音较多的副歌段
3. 气声、尾音、弱起较多的细腻段落

素材要求：

1. 尽量使用干净干声
2. 不要先上重混响
3. 不要使用已经明显失真的输入

## 3. 听感验收项目

每项按 `通过 / 勉强通过 / 不通过` 记录。

1. 音准  
判断点：主旋律是否稳定，长音是否飘，高音是否明显跑调。

2. 咬字  
判断点：歌词是否能听清，辅音是否糊成一团，快速段是否吞字。

3. 音色相似度  
判断点：是否已经接近目标音色，而不是只保留原唱音色。

4. 尾音与气声  
判断点：尾音是否自然收住，气声是否保留，是否出现机械断尾。

5. 连接与断句  
判断点：句与句之间是否有断裂、爆音、明显拼接痕迹。

6. 电音感与颗粒感  
判断点：是否有明显合成感、抖动感、颗粒噪声、金属味。

7. 底噪与杂音  
判断点：静音处是否有异常底噪，齿音区是否刺耳。

## 4. 对比测试顺序

建议固定同一段素材，按下面顺序对比：

1. 只加载主模型
2. 主模型 + 音质增强
3. 主模型 + 音质增强 + 音色增强

目标：

1. 第 2 组相对第 1 组，电音感和毛刺应明显下降
2. 第 3 组相对第 2 组，音色相似度应明显提升
3. 如果第 3 组咬字明显变差，说明音色增强比例可能过高

## 5. 判定标准

建议按下面标准判断：

1. 可用  
音准基本正常，能完整听完，无明显爆音和严重跑调。

2. 较好  
音色接近目标，咬字清楚，大部分句子自然，只有局部瑕疵。

3. 可交付  
整段听感稳定，音准、咬字、尾音、连接都没有明显短板，普通听众不会轻易察觉技术问题。

4. 接近最佳  
对比主模型版本有明确提升，音色、自然度、细节完整度都稳定，只有极少数高难段落还需手工微调。

## 6. 不通过时优先排查

1. 高音跑调  
先检查训练数据和输入素材，其次再看是否误开自动 F0。

2. 音色不像  
优先检查音色增强文件是否存在，再调 `cluster_ratio`。

3. 咬字差  
先降低 `cluster_ratio`，再复测。

4. 电音重  
优先确认音质增强模型是否加载成功，再看 `k_step` 是否过低。

5. 尾音断裂  
优先检查输入素材质量，再检查切片相关参数是否被手动改坏。
