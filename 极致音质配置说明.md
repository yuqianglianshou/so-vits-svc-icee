# 极致音质配置说明

## 📋 配置概览

项目已按照**极致音质配置**进行优化，默认仍保留当前稳定设置；同时新增了一条实验性的 HF ContentVec 路线，便于做 A/B 验证。

当前推荐的高质量默认值已统一维护在 [quality_presets.py](/Volumes/D/MyGitHub/so-vits-svc-icee/quality_presets.py)。
这意味着：

1. WebUI 和命令行推理使用同一套默认参数
2. 后续只需要维护一个预设源
3. Windows 验收时也能基于同一套参数判断结果

当前依赖安装也已统一为单一的 [requirements.txt](/Volumes/D/MyGitHub/so-vits-svc-icee/requirements.txt)：

1. Windows 会自动安装 `onnxruntime-gpu`
2. macOS 会自动处理 `setuptools` 兼容项，并跳过不适合本地 WebUI 的 `onnxsim`
3. 不再需要区分多个 requirements 文件

### ✅ 已应用的极致音质配置

```
编码器：vec768l12 (ContentVec 768维12层) - 固定
F0预测器：rmvpe (RMVPE) - 固定
浅层扩散：启用（k_step=200）- 默认启用
特征检索：启用（cr=0.5）- 默认启用
响度嵌入：启用 - 默认启用
二次编码：启用 - 默认启用
```

### 🧪 ContentVec 当前路线

当前项目已经完成 ContentVec 主线切换：

- `vec768l12`

它现在固定使用：

1. `Transformers / HF ContentVec`
2. 本地锁定目录 `pretrain/encoders/contentvec_hf/`
3. 必要时可回退到锁定的 Hugging Face 来源

也就是说，当前只保留一条 `vec768l12` 主线，底层统一走 HF / Transformers ContentVec。

## 🔧 代码修改说明

### 1. `train_pipeline/preprocess_flist_config.py` - 数据预处理配置

**修改内容：**
- ✅ 默认编码器保持为 `vec768l12`
- ✅ `vec768l12` 当前固定走 Transformers / HF ContentVec
- ✅ 默认启用响度嵌入（`--vol_aug` 默认 True）
- ✅ 根据编码器元信息自动设置 768 维特征维度

**使用方法：**
```bash
# 当前主路线（响度嵌入已默认启用）
python -m train_pipeline.preprocess_flist_config

# 如果需要禁用响度嵌入（不推荐）
python -m train_pipeline.preprocess_flist_config --no_vol_aug
```

**建议：**
- 日常训练继续使用 `vec768l12`
- 当前不再需要额外切换到其他 ContentVec 路线

---

### 2. `train_pipeline/preprocess_hubert_f0.py` - 特征提取

**修改内容：**
- ✅ 固定 F0 预测器为 `rmvpe`（移除其他 F0 预测器选项）
- ✅ 默认启用浅层扩散（`--use_diff` 默认 True）

**使用方法：**
```bash
# 简化后的命令（浅扩散已默认启用，使用 rmvpe）
python -m train_pipeline.preprocess_hubert_f0

# 如果需要禁用浅扩散（不推荐）
python -m train_pipeline.preprocess_hubert_f0 --use_diff=False

# 多进程加速（推荐）
python -m train_pipeline.preprocess_hubert_f0 --num_processes 8
```

---

### 3. `services/inference_main.py` - 推理脚本

**修改内容：**
- ✅ 固定 F0 预测器为 `rmvpe`
- ✅ 默认启用浅层扩散（`--shallow_diffusion` 默认 True）
- ✅ 默认 k_step=200（极致音质）
- ✅ 默认启用特征检索（`--feature_retrieval` 默认 True）
- ✅ 默认特征检索混合比例 cr=0.5
- ✅ 默认启用二次编码（`--second_encoding` 默认 True）
- ✅ 自动设置特征检索路径为 `logs/44k/feature_and_index.pkl`
- ✅ 使用浅扩散时自动禁用增强器

**使用方法：**
```bash
# 简化后的命令（所有极致音质选项已默认启用）
python -m services.inference_main \
  -m "logs/44k/G_30400.pth" \
  -c "configs/config.json" \
  -n "input.wav" \
  -s "speaker_name"

# 完整参数示例（大部分已默认，可省略）
python -m services.inference_main \
  -m "logs/44k/G_30400.pth" \
  -c "configs/config.json" \
  -n "input.wav" \
  -t 0 \
  -s "speaker_name" \
  -f0p rmvpe \          # 已固定，可省略
  -shd \                # 已默认启用，可省略
  -fr \                 # 已默认启用，可省略
  -dm "logs/44k/diffusion/model_0.pt" \
  -dc "configs/diffusion.yaml" \
  -ks 200 \             # 已默认200，可省略
  -se \                 # 已默认启用，可省略
  -cr 0.5               # 已默认0.5，可省略
```

---

### 4. `utils.py` - 工具函数

**修改内容：**
- ✅ `get_speech_encoder()`: 当前主线固定为 `vec768l12`（内部走 HF / Transformers 路线）
- ✅ 新增编码器元信息注册表，统一维护输出维度和加载方式
- ✅ `get_f0_predictor()`: 只保留 `rmvpe`，其他 F0 预测器自动切换为 `rmvpe`

**说明：**
- 如果代码中指定了未识别的编码器或 F0 预测器，会自动切换为默认稳定值
- 会输出警告信息提示已自动切换

## 🧪 当前 ContentVec 使用方式

当前不需要再单独切换实验编码器。只要使用默认：

- `vec768l12`

程序就会优先从：

- `pretrain/encoders/contentvec_hf/`

读取本地锁定的 HF ContentVec 模型；若本地目录缺失，才会回退到锁定的 Hugging Face 来源。

### 正常预处理与训练

```bash
python -m train_pipeline.preprocess_hubert_f0 --num_processes 8
python -m train_pipeline.train -c configs/config.json -m 44k
```

### 步骤 4：和稳定路线做 A/B 对比

建议至少对比这三项：

1. 发音跟随性
2. 咬字清晰度
3. 推理稳定性

### 步骤 5：随时回到稳定路线

只需要重新生成稳定配置即可：

```bash
python -m train_pipeline.preprocess_flist_config --speech_encoder vec768l12
```

然后重新做对应预处理与训练。

## ⚠️ 重要提示

### 1. 必须训练的模型
- ✅ **主模型**：必须训练
- ✅ **扩散模型**：必须训练（用于浅层扩散）
- ✅ **特征检索索引**：必须训练（用于特征检索）

### 2. 模型文件位置
确保以下文件存在：
- 主模型：`logs/44k/G_*.pth`
- 扩散模型：`logs/44k/diffusion/model_0.pt`
- 扩散配置：`configs/diffusion.yaml`
- 特征检索：`logs/44k/feature_and_index.pkl`

### 3. 预训练模型
确保已下载以下预训练模型：
- ContentVec：当前已锁定到本地 `pretrain/encoders/contentvec_hf/`
  - 来源：`lengyue233/content-vec-best`
  - 当前固定文件：`config.json`、`model.safetensors`
  - 建议长期优先使用本地目录，不要把线上仓库的最新状态当作唯一依赖
- RMVPE: `pretrain/encoders/rmvpe.pt`
- NSF-HiFiGAN: `pretrain/vocoders/nsf_hifigan/` (用于扩散模型)
- So-VITS 底模：`pretrain/base_models/44k/G_0.pth` 和 `pretrain/base_models/44k/D_0.pth`
- 扩散底模：`pretrain/base_models/44k/diffusion/model_0.pt`

训练开始时，程序会把底模自动同步到 `logs/44k/` 运行目录。

## 🎯 极致音质参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| 编码器 | vec768l12 | 最佳音质编码器 |
| F0预测器 | rmvpe | 最佳音高提取 |
| 浅层扩散 | 启用 | 提升音质，解决电音 |
| k_step | 200 | 扩散步数（越大音质越好） |
| 特征检索 | 启用 | 比聚类更好，咬字更清晰 |
| cr | 0.5 | 特征检索混合比例 |
| 响度嵌入 | 启用 | 匹配输入源响度 |
| 二次编码 | 启用 | 玄学选项，有时效果好 |
